# Day 4 - Loops

Today I learned one of the most important concepts in programming: **loops**.

At first, understanding how loops execute was a bit tricky,
 especially keeping track of the loop variable and knowing when the condition is checked.
 After solving several exercises, the execution flow became much clearer.

## Project Structure

```
Day4-Loops/
│

├── do_while/

└── README.md
```

## What I Learned

### for_loop

- Counting up and down
- Printing even numbers
- Multiplication tables
- Student attendance example

### while_loop

- Using a loop when the number of iterations isn't fixed
- Counting and printing even numbers with `while`

### do_while

- Understanding that the loop runs at least once before checking the condition
- Comparing `while` and `do...while`

### break_continue

- Using `break` to stop a loop early
- Using `continue` to skip the current iteration

### nested_loops

- Printing patterns
- Building a multiplication grid
- Understanding how one loop controls another

## What I Found Challenging

Nested loops were the most challenging part today. I had to think about which loop controls the rows and which controls the columns. Once I visualized the execution step by step, it became much easier.

## Key Takeaway

Loops make programs more efficient by avoiding repetitive code. More importantly, I learned to think about the execution flow instead of just memorizing the syntax.
