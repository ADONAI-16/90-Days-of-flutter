# Day 6 - Functions

Today I went deeper into Dart functions. I learned how to make functions more reusable by passing values, returning results, and using different types of parameters.

This was one of the most important days because I started writing functions that work together instead of writing everything inside `main()`.

## What I Learned

- Passing values using parameters
- Returning values with `return`
- Arrow functions (`=>`)
- Functions calling other functions
- Named parameters
- Required named parameters
- Optional positional parameters
- Default parameter values

## What I Practiced

- Built a simple calculator
- Generated student reports
- Created a login system
- Calculated employee bonuses
- Printed book information using named parameters
- Created accounts with default values

## What I Found Challenging

Understanding how one function can call another and use the returned value took some practice. Once I broke it down step by step, it became much clearer.

## Key Takeaway

Functions are not just for organizing code. They help separate responsibilities, reduce repetition, and make programs easier to understand and maintain.
